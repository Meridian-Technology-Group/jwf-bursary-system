# 28 — Pick reason codes (re-assessment)

Backlink: [[README#Re-assessment (year 2+)]]

On a re-assessment, reason codes are not optional — the recommendation
must explain why the payable fees changed (or stayed the same)
relative to the prior year.

## Prerequisites

- Re-assessment open on the **Recommendation** tab.
- Assessment status `COMPLETED`.

## Steps

1. Read the **Year-on-Year Comparison** panel (Assessment tab) before
   coming to the recommendation; identify which figures moved up or
   down and why.
2. On the **Recommendation** tab, expand the **Reason Codes** card.
3. Tick all codes that justify the year-on-year movement. Typical
   pairings:
   - Income up → codes in the **1–9: Income** bucket (e.g. "Significant
     pay rise", "New employment").
   - Income down → "Significant drop in earnings (illness, job loss)",
     "Self-employed business decline".
   - Property purchase / re-mortgage → **10–19: Property & Assets**
     codes.
   - New dependent / bereavement → **20–29: Family Circumstances**.
   - Honesty / credit concern raised → **30–39: Risk Flags**.
4. If nothing changed, still select at least one "no change"
   confirmation code (typically code 0/9 — confirm the exact code with
   your team's convention).
5. Save the recommendation.

## Verification

- The trigger row pill shows the count of selected codes.
- The export and PDF carry the full list of codes (number + label).
- The Reason Code Frequency report ([[35-reason-code-frequency]])
  reflects the new selections immediately for the round.

## Notes

- Deprecated codes do not appear in the picker but remain visible on
  recommendations that selected them previously.
- If the code you need does not exist, ask an admin to add it (see
  [[../admins/08-manage-reason-codes]]). Do not improvise via the
  free-text summary alone — the export grouping relies on codes.
